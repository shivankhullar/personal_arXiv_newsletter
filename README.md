## Configuration

The configuration file (`config.yaml`) controls all aspects of newsletter generation:

### Authors to Follow
```yaml
authors:
  - "Albert Einstein"
  - "Marie Curie"
```

Use the exact name as it appears in arXiv papers, or a close approximation. The system will fuzzy-match author names.

### arXiv Categories
```yaml
categories:
  - "astro-ph.CO"  # Cosmology and Nongalactic Astrophysics
  - "astro-ph.GA"  # Astrophysics of Galaxies
  - "cs.AI"        # Artificial Intelligence
  - "physics.bio-ph" # Biological Physics
```

Find category codes at: https://arxiv.org/category_taxonomy

### Keywords
```yaml
keywords:
  - "machine learning"
  - "gravitational waves"
  - "quantum computing"
```

Papers matching these keywords in titles or abstracts will be included.

### Search Parameters
```yaml
days_back: 7              # How many days to look back
max_papers: 20            # Maximum papers in newsletter
min_similarity_score: 0.3 # Minimum relevance score (0-1)
```

### Output Settings
```yaml
output:
  directory: "newsletters"
  filename: "arxiv_newsletter_{date}.pdf"
  include_abstracts: true
  include_links: true
  group_by_category: true
```

### Advanced Settings
```yaml
advanced:
  use_semantic_similarity: true  # Use ML-based similarity
  reference_papers_limit: 50     # Papers used for similarity
  author_weight: 0.6             # Balance author vs. content (0-1)
```

## Usage Examples

### Basic Usage
```bash
# Generate newsletter with default config
arxiv-newsletter

# Use custom config file
arxiv-newsletter --config my_config.yaml

# Specify output location
arxiv-newsletter --output ~/Desktop/newsletter.pdf
```

### Advanced Options
```bash
# Look back 14 days instead of configured value
arxiv-newsletter --days 14

# Limit to 10 papers
arxiv-newsletter --max-papers 10

# Generate HTML preview instead of PDF
arxiv-newsletter --html

# Disable semantic similarity (faster)
arxiv-newsletter --no-similarity

# Verbose output
arxiv-newsletter --verbose
```

### Cache Management
```bash
# Check cache status
arxiv-newsletter --cache-info

# Clear cache (force fresh fetch next time)
arxiv-newsletter --clear-cache

# Ignore cache and fetch fresh data
arxiv-newsletter --no-cache

# Fast PDF regeneration from cached data (no API calls)
# Perfect for iterating on PDF styling!
arxiv-newsletter --pdf-only
```

The caching system automatically stores:
- Fetched papers from arXiv
- Reference papers for similarity
- Filtered and scored results

Cache is valid for 24 hours and automatically invalidates when you change:
- Authors to follow
- Categories or keywords
- Exclusion criteria
- Days back setting

## How It Works

The newsletter generator follows these steps:

1. **Fetch Papers**: Retrieves recent papers from arXiv based on:
   - Papers by authors you follow
   - Papers in categories you specified
   - Papers matching your keywords

2. **Compute Similarity**: If enabled, fetches reference papers from your followed authors and uses TF-IDF to compute semantic similarity between candidate papers and reference papers.

3. **Filter and Rank**: Scores each paper based on:
   - Author match (weighted by `author_weight`)
   - Keyword matches
   - Semantic similarity (weighted by `1 - author_weight`)
   
   Papers are ranked by score and filtered by `min_similarity_score`.

4. **Generate Newsletter**: Creates a formatted PDF (or HTML) with:
   - Paper titles and authors
   - Publication dates and categories
   - Match reasons and relevance scores
   - Abstracts (optional)
   - Links to arXiv and PDF (optional)

## Troubleshooting

### No papers found
- Check that your authors, categories, and keywords are correct
- Try increasing `days_back` in config
- Verify arXiv category codes at https://arxiv.org/category_taxonomy

### No papers pass threshold
- Lower `min_similarity_score` in config
- Disable semantic similarity with `--no-similarity`
- Increase `max_papers` limit

### ImportError or ModuleNotFoundError
Make sure all dependencies are installed:
```bash
pip install -e .
```

### PDF generation fails
Ensure reportlab is properly installed:
```bash
pip install --upgrade reportlab
```
